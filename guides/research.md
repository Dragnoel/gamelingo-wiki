# Research and citation discipline — Gaming Language Wiki

## Source hierarchy
1. Peer-reviewed publications (raw/papers/)
2. Primary sources: in-game text, official manuals, designer transcripts (raw/manuals/, raw/transcripts/)
3. Reputable secondary writing (named author, dated, raw/articles/)
4. Wiki/forum — pointer to primary only; never the citation itself

## Citation form
- [^smith-2024-p12]: Smith, J. (2024). *Title*. Publisher, p. 12. raw/papers/smith-2024.pdf.
- Cite to the page or section, not just the work.
- Unlocatable source: {{unverified}} + add to wiki/_review.md.

## Conflicting sources
- Do not silently pick a winner. State both positions, note unresolved status.
- Add to [[Disputed claims]] index page.
- Do not assert resolution until a human reviews and clears from _review.md.

## Never in a wiki page
- Agent's own opinion or inference beyond stated sources.
- Speculation about player intent, designer motivation, or cultural meaning without a sourced claim.
- Training-data knowledge used as a citation — all claims must trace to a file in raw/.
- Verbatim quotes longer than 15 words.
- More than one verbatim quote per source per page.

## Source page discipline
- Every file added to raw/ should have a corresponding wiki/source/ page created during /ingest.
- Source pages summarise the file, list key terms and claims found, and record the bib key.
- Source pages are not analysis — they describe what the file contains, not what it means.
