# /cleanup — Gaming Language Wiki

## Purpose
Propose file deletions for human review. Never delete files directly.
Output a shell script to queries/_cleanup.sh. The human reviews and runs it.

## Procedure

### 1. Identify candidates for deletion
Common cases:
- wiki/terms/ files without a corresponding entries/ JSON (orphan renders — re-run render_entries.py instead of deleting)
- wiki/source/ pages for raw/ files that have been removed
- Duplicate or superseded wiki/ pages
- Empty placeholder files that were never filled in

### 2. Write queries/_cleanup.sh
Format strictly:
```bash
#!/usr/bin/env bash
set -euo pipefail

# Review each line before running. This script was generated by /cleanup on YYYY-MM-DD.
# Run from the vault root: bash queries/_cleanup.sh

rm "wiki/source/old-source.md"   # orphan: raw/papers/old-source.pdf no longer exists
```

One rm per line. Always double-quoted paths. Always a comment explaining why.
Do not include rm commands for entries/ JSON files without explicit human instruction.
Do not include rm -rf under any circumstances.

### 3. Update PROJECT_STATUS.md
Note that a cleanup script was written and is pending human review.

## Hard prohibitions during /cleanup
- Do not run rm, mv, or any deletion command.
- Do not delete entries/ JSON files.
- Do not overwrite an existing queries/_cleanup.sh without noting the previous one was superseded.
